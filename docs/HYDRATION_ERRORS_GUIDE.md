# Руководство по устранению и предотвращению ошибок гидратации (React Hydration Errors)

Это руководство создано по итогам решения сложных ошибок гидратации (`Text content does not match server-rendered HTML` и `Hydration failed because the initial UI does not match what was rendered on the server`) в проекте PharmaTurk.

## 1. Проблема сортировки строк: `localeCompare`
Одной из самых трудноуловимых причин ошибок гидратации была различная сортировка русских слов на сервере Node.js и в клиентском браузере (Chrome / Safari). 

**Причина:** 
Если применять `a.name.localeCompare(b.name)` без указания локали, сервер может сортировать кириллические символы иначе, чем браузер, из-за различных системных локалей или отсутствия полных пакетов ICU в Docker Alpine. Из-за этого массивы отрендеренных элементов возвращались в разном порядке, вызывая ошибку `Prop did not match. Server: filter-item-cat-43 Client: filter-item-cat-47`.

**Правило на будущее:**
Всегда в явном виде передавайте параметр локали (например, `'ru'`) при использовании сортировки строк в SSR-компонентах.
```typescript
// НЕПРАВИЛЬНО (Вызывает ошибку гидратации):
categories.sort((a, b) => a.name.localeCompare(b.name))

// ПРАВИЛЬНО:
categories.sort((a, b) => a.name.localeCompare(b.name, 'ru'))
```

## 2. Проблема перевода и отложенного рендеринга (`isMounted`)
Для борьбы с ошибками гидратации часто применялся хак с проверкой состояния `isMounted`, чтобы на сервере отрендерить нелокализованное имя из базы данных, а на клиенте подменить его на перевод.

**Причина:** 
Сервер отправлял HTML со строкой "Кеды" (из БД), а клиент при первой отрисовке заменял её на "Keds" (перевод). С точки зрения React, первый серверный слепок HTML должен **полностью** совпадать с первым клиентским слепком HTML. Иначе выбрасывается `Text content does not match`.

**Правило на будущее:**
Используйте функцию `t()` напрямую как на сервере, так и на клиенте. `next-i18next` корректно обрабатывает переводы внутри функции `getServerSideProps` и передает их в виде стартовых пропсов, гарантируя, что обе стороны отрендерят одинаковый текст с долей секунды.

```typescript
// НЕПРАВИЛЬНО (Вызывает ошибку гидратации):
const label = isMounted ? getLocalizedCategoryName(...) : rawDbName

// ПРАВИЛЬНО:
const label = getLocalizedCategoryName(...)
```

## 3. Добавление новых подкатегорий и атрибутов 
При добавлении новых подкатегорий или опций фильтров из админ-панели (например: "Зимние куртки") их русские имена пробрасываются в приложение в виде слагов (slug).

**Правило на будущее:** 
Для того чтобы всё переводилось корректно без падений, необходимо добавлять сгенерированный ключ в конфигурацию `frontend/public/locales/en/common.json`. 

Пример добавления перевода:
Система берет slug (например `winter-jackets`) и формирует ключ `category_winter-jackets_name`. Вам нужно просто добавить его в английский JSON:
```json
"category_winter-jackets_name": "Winter Jackets"
```
Для значений атрибутов (таких как "Кожа" или "Кеды") ключи формируются как `attr_val_[значение в нижнем регистре с нижними подчеркиваниями]`:
```json
"attr_val_зимние_куртки": "Winter Jackets"
```
Если английский ключ не будет найден (например, забыли внести), React безопасно отрендерит `fallback` (исходное русское имя) и на сервере, и на клиенте, **без ошибок гидратации**.
