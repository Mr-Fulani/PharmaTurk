# Руководство по массовым действиям в админке каталога

Документ описывает, какие массовые действия доступны в карточках товаров и услуг каталога, какой набор считается общим и как добавлять доменные действия без расхождений.

---

## Главное правило

Для товарных и сервисных админок меню массовых действий теперь должно собираться централизованно. Это убирает случайные расхождения между категориями, когда одинаковые действия раньше задавались вручную в разных `ModelAdmin`.

Норма такая:

- сначала идут общие действия, доступные всем;
- затем идут общие AI-действия, если админка их поддерживает;
- затем идут доменные действия, если они действительно нужны конкретной категории;
- системное действие удаления (`delete_selected`) остаётся доступным, но идёт после доменных действий.

---

## Базовый набор для большинства товарных категорий

Эти действия должны быть одинаковыми у категорий, работающих через общий AI-каркас:

1. `[Общее] Сделать активными`
2. `[Общее] Сделать неактивными`
3. `[AI] Полная AI обработка (без авто-применения)`
4. `[AI] Полная AI обработка + авто-применение`
5. `[Общее] Поиск кандидатов в дубликаты (на модерацию)`

К этому набору относятся, например:

- одежда;
- обувь;
- электроника;
- мебель;
- украшения;
- книги;
- головные уборы;
- нижнее бельё;
- исламская одежда;
- базовый каталог `Product`.

---

## Домены с медиа-обогащением

Для медикаментов и БАДов к общему набору добавляется доменное действие медиа-обогащения:

1. `[Общее] Сделать активными`
2. `[Общее] Сделать неактивными`
3. `[AI] Полная AI обработка (без авто-применения)`
4. `[AI] Полная AI обработка + авто-применение`
5. `[Общее] Поиск кандидатов в дубликаты (на модерацию)`
6. `[Категория] Обогатить медиа (картинки)`

Почему так:

- это домены, где ручной запуск подтягивания картинок является штатной операцией редактора;
- доменное действие не заменяет общий набор, а идёт после него.

---

## Парфюмерия

Парфюмерия больше не держит отдельный набор массовых действий.

Теперь она использует тот же общий набор, что и остальные товарные категории:

1. `[Общее] Сделать активными`
2. `[Общее] Сделать неактивными`
3. `[AI] Полная AI обработка (без авто-применения)`
4. `[AI] Полная AI обработка + авто-применение`
5. `[Общее] Поиск кандидатов в дубликаты (на модерацию)`

Действие `Отметить как бестселлер` убрано из массового меню как неуниверсальное и слабополезное для bulk-операций.

---

## Услуги

Для услуг подключён общий безопасный минимум:

1. `[Общее] Сделать активными`
2. `[Общее] Сделать неактивными`

AI-действия и действия по дублям в `ServiceAdmin` не добавляются автоматически, потому что это отдельный контур, не совпадающий с товарным pipeline.

---

## Важное замечание про поиск дублей

Действие `Поиск кандидатов в дубликаты (на модерацию)` работает не по выбранным строкам, а по всему каталогу.

Оно не объединяет товары автоматически.

Теперь результатом становятся карточки кандидатов в разделе модерации дубликатов.

## Варианты товаров

Для вариантных админок порядок такой:

1. действия активации и деактивации варианта;
2. AI-действия для вариантов;
3. системные действия Django.

Это означает, что редактор сначала видит самые частые операционные действия, а затем уже AI-инструменты.

---

## Как различать общие и доменные действия

В стандартном выпадающем списке Django Admin надёжно раскрашивать отдельные пункты по цветам нельзя без кастомизации шаблонов и JavaScript, которую легко сломать обновлениями.

Поэтому используется безопасная альтернатива:

1. префикс `[Общее]` для глобальных действий;
2. префикс `[AI]` для общих AI-операций;
3. префикс `[Категория]` для доменных действий.

---

## Как расширять меню правильно

Если нужно добавить новое массовое действие:

1. Сначала определить, это общее действие для большинства категорий или доменное исключение.
2. Если действие общее, добавить его в централизованный порядок action-меню.
3. Если действие доменное, явно объявить его в конкретной админке как локальное расширение.
4. Обязательно добавить тест на состав и порядок меню.

Нельзя:

- вручную раскидывать одинаковые списки `actions` по нескольким файлам;
- менять порядок действий только в одной категории без причины;
- добавлять глобальные действия так, будто они работают только по выбранным строкам.

---

## Где проверять после изменений

- `backend/apps/catalog/admin_base.py` — общий механизм сборки и порядка действий;
- `backend/apps/catalog/admin.py` — крупные товарные админки;
- `backend/apps/catalog/admin_wave2.py` — простые домены и домены с медиа-обогащением;
- `backend/apps/catalog/admin_books.py` — книги;
- `backend/apps/catalog/admin_perfumery.py` — парфюмерия;
- `backend/apps/catalog/tests/test_admin_actions.py` — регрессионные тесты на меню действий.
